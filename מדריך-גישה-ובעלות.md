# מדריך גישה ובעלות — איך לוקחים שליטה על האתר

מדריך זה מסביר איך **גלי (או חבר טכני מטעמה)** מקבלים גישה מלאה לאתר
`galigeula.com` — כדי לערוך, לפרסם, ולנהל הכול באופן עצמאי.
אפשר לשלוח את המסמך הזה ישירות לחבר שעוזר.

---

## מפת השטח — ממה האתר מורכב

| רכיב | מה זה | מי מחזיק כרגע |
|------|-------|----------------|
| **מאגר הקוד (GitHub)** | כל הקוד והתוכן | חשבון GitHub של דיויד: `silverdavi/gali-cohen.com` |
| **אירוח (GitHub Pages)** | מפרסם את האתר, בחינם | מוגדר בתוך אותו מאגר |
| **דומיין** `galigeula.com` | הכתובת | AWS Route 53, חשבון של דיויד |
| **מערכת ניהול תוכן** `/admin` | עריכה בלי קוד | רץ מתוך המאגר |

> **השורה התחתונה:** מי שיש לו גישת כתיבה למאגר ב‑GitHub — שולט בתוכן ובפרסום.
> הדומיין הוא הדבר היחיד שנמצא בנפרד (ב‑AWS).

## שלב 1 — לפתוח חשבון GitHub (אם אין)

1. נכנסים ל‑https://github.com → **Sign up**.
2. בוחרים שם משתמש (למשל `galigeula`), אימייל, סיסמה.
3. מאשרים את האימייל.
4. מומלץ מאוד להפעיל **אימות דו‑שלבי (2FA)** ב‑Settings → Password and
   authentication.

📌 שולחים לדיויד את **שם המשתמש ב‑GitHub** — הוא צריך אותו לשלב הבא.

---

## שלב 2 — לקבל גישה למאגר (אחת משתי דרכים)

### דרך א׳ — להתווסף כשותף עריכה (הכי מהיר)
דיויד עושה: במאגר → **Settings → Collaborators → Add people** → מוסיף את שם
המשתמש שלך עם הרשאת **Write** (או **Admin** לשליטה מלאה).
מקבלים הזמנה במייל → לוחצים **Accept**. זהו — יש גישה.

### דרך ב׳ — להעביר בעלות מלאה על המאגר (Transfer)
אם רוצים שהאתר יהיה לגמרי תחת החשבון של גלי:
דיויד עושה: במאגר → **Settings → General → Danger Zone → Transfer
ownership** → מקליד את שם המשתמש של גלי.
אחרי ההעברה, הכתובת תהיה `גלי/gali-cohen.com`, וגלי הבעלים המלא.

> אחרי דרך ב׳ צריך לעדכן את `repo:` בקובץ `public/admin/config.yml`
> לשם הבעלים החדש, אחרת מסך ה‑`/admin` ימשיך להצביע על המאגר הישן.

## שלב 3 — לערוך בלי קוד (CMS), היומיומי

1. נכנסים ל‑**https://galigeula.com/admin**
2. **Sign in with Token** → נפתח GitHub ליצירת טוקן (Personal Access Token):
   - GitHub → **Settings → Developer settings → Personal access tokens →
     Tokens (classic) → Generate new token**.
   - הרשאה נדרשת: סימון **`repo`**. נותנים תוקף (למשל שנה), יוצרים, **מעתיקים**.
   - חוזרים ל‑`/admin` ומדביקים. הדפדפן זוכר את הטוקן.
3. עורכים → **Save**. האתר מתפרסם לבד תוך דקה–שתיים.

> הטוקן = הסיסמה לעריכה. לא משתפים אותו. אם דלף — מבטלים אותו ב‑GitHub
> ויוצרים חדש.

## שלב 4 — לערוך קוד (לחבר הטכני)

```bash
# מורידים את הקוד
git clone https://github.com/silverdavi/gali-cohen.com.git
cd gali-cohen.com/site

# מתקינים ומריצים מקומית
npm install
npm run dev          # נפתח ב‑http://localhost:5173

# אחרי שינוי — שומרים ומעלים
git add -A
git commit -m "תיאור השינוי"
git push
```

כל `push` ל‑ענף `main` מפעיל פרסום אוטומטי (GitHub Actions → GitHub Pages).
רואים את מצב הפרסום בלשונית **Actions** במאגר.

**מבנה מהיר:**
- `site/src/content/*.yaml` — כל הטקסטים והתוכן (עברית).
- `site/src/index.css` — עיצוב (צבעים/גופנים בראש הקובץ).
- `site/src/components/*.tsx` — חלקי האתר.

## שלב 5 — הדומיין (galigeula.com)

הדומיין מנוהל ב‑**AWS Route 53** בחשבון של דיויד. שתי אפשרויות:

- **להשאיר אצל דיויד** — הכי פשוט. ה‑DNS כבר מכוון ל‑GitHub Pages; אין מה
  לעשות. עריכת התוכן והקוד עצמאית לגמרי גם ככה.
- **להעביר בעלות על הדומיין** — אם רוצים שגם הדומיין יהיה תחת גלי: צריך
  להעביר את ה‑domain בין חשבונות AWS (תהליך של AWS), או לרכוש/לנהל אותו
  אצל רשם דומיינים אחר ולכוון את ה‑DNS ל‑GitHub Pages. מורכב יותר — לתאם
  עם דיויד.

> כדי שהאתר יעבוד על הדומיין, ב‑GitHub: **Settings → Pages → Custom domain**
> חייב להישאר `galigeula.com` (וקובץ `site/public/CNAME` מכיל אותו).

## סיכום — מה לשלוח לדיויד כדי לקבל שליטה מלאה

1. **שם המשתמש שלך ב‑GitHub** (לשלב 2 — שותף או העברת בעלות).
2. להחליט: גישת שותף (מהיר) או העברת בעלות מלאה (עצמאות מלאה).
3. להחליט לגבי הדומיין: להשאיר אצל דיויד (מומלץ לבינתיים) או להעביר.

אחרי זה — כל העריכה היומיומית נעשית לבד דרך `galigeula.com/admin`.
